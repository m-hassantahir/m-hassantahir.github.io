Toy Store Performance Analysis

May 27, 2024

Project Overview

This project involved designing a data-driven Power BI dashboard for a toy company to analyze business performance across sales, product, and store insights. The goal was to transform raw business data into interactive visualizations that aid decision-making. The dashboard consists of four pages:

  1. Sales Performance: High-level overview of sales metrics, trends, and comparisons.
  2. Product Performance: Analysis of individual product performance in terms of sales, revenue, and profitability.
  3. Store Performance: Visualization of store-level performance, including revenue, YoY differences, and inventory.
  4. Tooltip Page: Additional contextual information displayed interactively across the dashboard.

The project covered the entire process from data preparation to dashboard design, including ETL processes, DAX measures, and interactive visualizations.

Key Features

  1. Interactive Dashboards: Four pages with dynamic visualizations for sales, product, and store performance.
  2. Custom Layout and Design: Designed in PowerPoint, imported into Power BI, with a consistent color palette and branding.
  3. Advanced DAX Measures: Created complex calculations like YTD Revenue, YoY Difference, Running Total Revenue, and Progress Towards Goal.
  4. Data Visualizations: Used line charts, bar charts, matrices, maps, and cards to display key metrics.
  5. Tooltips: Custom tooltips for additional context when hovering over visuals.
  6. Conditional Formatting: Applied to highlight key metrics and improve readability.
  7. Responsive Design: Ensured the dashboard is interactive and works seamlessly across devices.
  8. Data Modeling: Established relationships between tables (e.g., Sales, Product, Store, Dates) for accurate analysis.

Technical Details

1. Data Preparation:

  • Data Sources: Five main tables (Dates, Sales, Store, Product, Inventory).
  • ETL Process: Used Power Query Editor to clean, transform, and merge data.
  • Data Modeling: Created relationships between tables (e.g., Dates ↔ Sales, Product ↔ Sales).

2. DAX Measures:

  • Basic Measures: Total Revenue, Cost, Units Sold, Profit.
  • Advanced Measures: YTD Revenue, Previous YTD Revenue, YoY Difference, Running Total Revenue, Progress Towards Goal.
  • Custom Columns: Profit Margin, Revenue Per Unit.

3. Visualizations:

  • Line Charts: Revenue trends over time.
  • Bar Charts: YoY comparisons by product and store.
  • Matrix: Tabular data for YTD Revenue, YTD Profit, and YoY Difference.
  • Maps: Geographic representation of store performance.
  • Cards: Key metrics like Revenue, Profit, and Goal Progress.

4. Design and Layout:

  • Custom Themes: Imported color palettes and layouts from PowerPoint.
  • Background Design: High-quality SVG files for consistent branding.
  • Tooltips: Customized to display additional information interactively.

5. Interactivity:

  • Filters and Slicers: Enabled dynamic filtering by time, product, and store.
  • Dynamic Measures: Data updates based on user interactions.

Impact

2. Enhanced Data Accessibility:

  • Interactive visualizations made complex data easy to understand for non-technical users.
  • Example: Tooltips provided additional context without cluttering the main visuals.

3. Time Efficiency:

  • Automated data preparation and visualization reduced manual effort and improved reporting speed.
  • Example: YTD and YoY calculations were automated, eliminating the need for manual updates.

4. Consistent Branding:

  • Custom themes and layouts ensured the dashboard aligned with the company's branding.
  • Example: Imported company colors and logos for a professional look.

5. Scalability:

  • The dashboard was designed to handle additional data and metrics as the business grows.
  • Example: Dynamic measures and filters allowed for easy expansion of the dataset.
