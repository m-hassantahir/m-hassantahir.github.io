Project Performance & Work Hours Analysis

January 20, 2025

Project Overview

  • Project Title: Power BI Dashboards for Project Performance and Work Hours Analysis
  • Objective: To create interactive Power BI dashboards that track project performance and manage working hours, focusing on monitoring estimated versus actual hours with dynamic filtering for detailed analysis.
  • Client Needs: The client required a solution to effectively monitor project performance and working hours, with the ability to update the dashboards weekly with revised data.
  • Data Sources: Primavera and Excel (wide-format Excel workbook).
  • Outcome: Developed two interactive dashboards that allowed the client to monitor project performance, analyze working hours, and explore various scenarios efficiently.

Key Features

  • Dynamic Filtering: Incorporated slicers for filtering data by company, scope of work, and dates, enabling users to analyze performance across various scenarios (e.g., year, quarter, or month).
  • Interactive Dashboards: Designed two dashboards:
    • Line and Clustered Column Chart Dashboard: Visualized the project's overall S-curve with metrics like planned, earned, and forecast percentages.
    • Work Hours and Employee Analysis Dashboard: Displayed working hours over time, number of employees, and average sell rate.
  • Data Normalization: Split the main data table into smaller tables to eliminate redundancy and improve performance.
  • Weekly Updates: Structured the data model for easy integration of new Excel data, allowing for weekly updates with minimal effort.

Technical Details

  • Data Transformation:
    • Unpivoting Data: Converted wide-format data into long-format using Power Query's Unpivot Columns feature.
    • Transposing Data: Rearranged rows and columns to have dates in rows and values in columns.
    • Data Cleaning: Removed duplicates, inconsistencies, and unnecessary fields.
  • Data Modeling:
    • Normalization: Split the main table into smaller tables (e.g., employee details, task categories, and dates) to reduce redundancy.
    • Relationships: Created logical relationships between tables using the dates column as a primary key.
  • Visualization:
    • Line and Clustered Column Chart: Used to visualize the project's S-curve.
    • Line Chart: Displayed working hours over time.
    • Cards: Showed KPIs like the number of employees and their average sell rate.
  • Tools: Power BI, Power Query, Excel.

Skills

  • Power BI Expertise: Dashboard creation, data visualization, Power Query, and advanced filtering.
  • Data Modeling: Normalization, relationship building, and optimization.
  • Data Transformation: Unpivoting, transposing, and cleaning data for analysis.
  • Project Management Analysis: Ability to track and compare project metrics like planned vs. actual working hours.
  • Problem Solving: Devised multiple solutions to handle wide-format data and optimize performance for large datasets.
  • Communication: Designed dashboards with clear, actionable insights tailored to the client's needs.

Impact

  • Improved Decision-Making: The dashboards provided real-time insights into project performance and working hours, enabling the client to make informed decisions.
  • Enhanced Efficiency: The dynamic filtering and interactive features allowed users to drill down into detailed metrics, improving the efficiency of project analysis.
  • Scalability: The normalized data model and structured approach allowed for easy weekly updates, ensuring the solution was scalable and adaptable to future needs.
  • Client Satisfaction: The user-friendly dashboards met the client's requirements and provided a clear, actionable view of project performance and working hours.
