Sales Performance Dashboard

January 25, 2024

1. Project Overview

Description:

This project involved creating a Sales Tracking Dashboard for Compsee Ecommerce using Microsoft Power BI. The goal was to provide actionable insights into the company's sales performance by visualizing key metrics, trends, and patterns. The dashboard is interactive, allowing users to explore data dynamically using filters and slicers. It includes a variety of visualizations such as column charts, pie charts, and bar charts, along with key metrics displayed via cards.

Purpose:

The dashboard was designed to help Compsee Ecommerce monitor sales performance, identify trends, and make data-driven decisions. It provides a comprehensive view of sales data, including profit by month, quantity by category, and top-performing customers.

2. Key Features

  1. Interactive Visualizations:
    • Column Charts: To visualize profit trends over time.
    • Pie Charts: To show the distribution of quantities by category and payment modes.
    • Stacked Bar Charts: To display profit contributions by sub-category and top customers.
  2. Dynamic Filters:
    • Quarter Filter: Allows users to filter data by quarter.
    • State-Based Filter: Enables filtering by state for regional analysis.
  3. Key Metrics Display:
    • Cards: Displaying Total Amount, Quantity, Profit, and Average Order Value (AOV).
  4. Conditional Formatting:
    • Applied to highlight positive and negative values, making it easier to interpret data at a glance.
  5. Custom Metrics:
    • Average Order Value (AOV): Calculated as Amount divided by Quantity.
    • Top N Filters: Used to highlight top-performing categories, customers, and statuses.
  6. Polished Layout:
    • A professional and cohesive design with consistent formatting and styling.

3. Technical Details

The project involved the following technical steps:

  1. Data Import:
    • Imported two CSV files: Orders Table and Order Details Table using Power BI's Get Data feature.
  2. Data Cleaning and Transformation:
    • Used Power Query to clean and transform the data:
    • Changed data types (e.g., text to date).
    • Removed unnecessary columns.
    • Split columns for better organization.
    • Removed duplicates.
    • Created custom columns (e.g., AOV).
    • Aggregated data using Group By.
  3. Data Modeling:
    • Established relationships between the Orders Table and Order Details Table in the Model View.
    • Defined cardinality as One-to-Many.
    • Ensured active relationships for seamless data integration.
  4. Dashboard Creation:
    • Designed visualizations such as column charts, pie charts, and bar charts.
    • Added interactive slicers for quarters and states.
    • Applied conditional formatting for better data interpretation.
  5. Custom Calculations:
    • Used DAX (Data Analysis Expressions) to create calculated columns and measures, such as AOV.
  6. Formatting and Customization:
    • Used the Format Painter to maintain consistency across visualizations.
    • Added a canvas background image and adjusted transparency for a polished look.
